Chapter 2: Caffeine, Jet Lag, and Melatonin - Losing and Gaining Control of Your Sleep Rhythm
  • There are two main factors that determine when you want to sleep and when you want to be awake:
    • The first factor is a signal beamed out from your internal twenty-four-hour clock located deep within your brain.
      • Your twenty-four-hour clock is called your circadian rhythm.
      • The tempo helps to determine when you want to be awake and when you want to be asleep, but it controls other rhythmic patterns too.
      • The circadian rhythm does not come from a night and day cycle. Humans (and other plants and animals) have an internally generated circadian rhythm.
      • Humans' cycle of wake and sleep were not precisely twenty-four hours in length.
        • The light of the sun methodically resets our inaccurate internal timepiece each and every day, "winding" us back to precisely, not approximately, twenty-four hours.
        • The brain can also use other external cues for the purpose of biological clock resetting, as long as they are reliably repeating.
      • Although every human being displays an unyielding twenty-four-hour pattern, the respective peak and trough points are strikingly different from one individual to the next.
        • There are morning types. Makes up 40% of the populace. Wakes up at or around dawn and function optimally at this time of day.
        • There are evening types. Makes up 30% of the population. Prefer to go to bed late and subsequently wake up late the following morning or afternoon.
        • The remaining 30% lie somewhere in between morning and evening types, with a slight leaning toward eveningness.
      • We live in a society that is strongly biased toward early start times that punish night owls and favor morning larks.
      • The brain communicates its repeating signal of night and day to your brain and body using a circulating messenger called melatonin.
      • Melatonin helps regulate the timing of when sleep occurs, but it has little influence on the generation of sleep itself.
      • Melatonin is not a powerful sleeping aid in and of itself.
      • For every day you are in a different time zone, your suprachiasmatic nucleus (the part of your brain that uses melatonin) can only readjust by about one hour.
        • For every hour the time zone differs by, you need an additional day to recover your natural melatonin schedule.
      • Jet lag places a torturous physiological strain on the brain, and a deep biological stress upon the cells, organs, and major systems of the body.
    • The second factor is a chemical substance that builds up in your brain and creates a "sleep pressure".
      • This chemical is called adenosine, which builds in your brain. The longer you are awake, the more adenosine will accumulate.
      • You can artificially mute the sleep signal of adenosine by using a chemical that makes you feel more alert and awake: caffeine.
      • Levels of circulating caffeine peak approximately thirty minutes after oral administration.
      • Caffeine has an average half-life of five to seven hours (after 5 to 7 hours, 50% of the caffeine will still be circulating in your system).
      • One cup of tea or coffee in the morning will last much of the day, and should they have a second cup, even early in the afternoon, they will find it difficult to fall asleep in the evening.
      • The older we are, the longer it takes our brain and body to remove caffeine.
      • Caffeine is the only addictive substance that we readily give to our children and teens.
      • After approximately eight hours of healthy sleep in an adult, your adenosine levels are reset back to 0.
        • If you don't get enough sleep, some quantity of yesterday's adenosine remains, the sleep debt slowly building up.
    • The two factors, adeosine and circadian rhythm, are two distinct and separate systems that are ignorant of each other.

Extensive List of Docker Commands

Basic Information
  • Docker is a process, to be used in Shell, that lets us run multiple containers from different available images.
  • What is an image? It is the source code, binaries, and libraries of an application.
  • What is container? It is an instance of the image as a process.

Docker Container Commands

Side Note: To get a list of commands and/or modifications to commands, enter (command) --help
Example: docker container --help, docker container start --help, docker image --help, etc
Docker Container Run
This is the most basic and important Docker Container commands. It can be executed in many ways depending on what is added to the command.
  • docker container run (name of image) : runs a container based on the image.
    • Side Note: docker container run must be at the beginning of the command. The image specification must be at the end of the command. Any additional modifications are added in the middle of the command
Modifications
REMINDER: Modifications must be added between 'docker container run' and the name of the image
  • -it: starts a container interactively
  • -d: runs a container in the background (d stands for daemon)
  • --name: names the created container
  • -p HOST POST:CONTAINER PORT: publishes the container where the CONTAINER PORT is the port at which the container listens and the HOST PORT is the port where a user can access the container
  • -e: sets environment variables
  • -v: bind mounts a volume

Other Docker Container Commands

Side Note: Some commands have additional modifications. To get an extensive list, you can always add --help to the end of any command
  • docker container start (id of container): starts a stopped container
    • Side Note: You do not need to put in the full id of the container. You only need to put in the first 3 characters or more, simply making sure the ID is unique
  • docker container stop (id of container): stops the container from running
  • docker container rm (list of container id's): removes all stopped containers within the list of container id's, running containers will not be removed and you will be informed that they could not be removed
    • Side Note: adding '-f', force, after rm, will remove stopped and running containers, removing all the containers within the list
    • Side Note: docker rm $(docker ps -a -q) removes all stopped containers
    • Side Note: Similarly, docker rm -f $(docker ps -a -q) removes all containers
  • docker container ls: lists all running/active containers
    • Side Note: docker ps also achieves the same result
  • docker container ls -a: list all containers (including stopped containers)
    • Side Note: docker ps -a also achieves the same result
  • docker container logs (name of container): shows logs for a specific container
  • docker container top (name of container): shows processes for the container (includes both master and worker processes)
  • docker container inspect (name of container): shows the metadata about the container (start-up config, volumes, networks, etc)
    • Side Note: adding '--format*' after 'inspect' formats the output of commands using "Go templates"
  • docker container stats (name of container): repeatedly displays the performance stats of all running containers on regular intervals

Docker Network Commands

  • docker network inspect (name of network): gives detailed information about the network
  • docker network create (name of network): creates a new network
  • docker network connect (network id) (container id): connects a container to a network
  • docker network disconnect (network id) (container id): disconnects a container from a network

Docker Image Commands

  • docker image history (name): shows layers of changes made in image
  • docker image inspect (name): returns JSON metadata about the image
  • docker image tag (image name) (tag): places a tag onto the image
    • docker image tag (REPOSITORY) (NEWREPOSITORY): makes a new image with a different repository (NEWREPOSITORY), same image id, same tag
    • docker image tag (REPOSITORY) (REPOSITORY:TAG): makes a new image with the same repository, same image id, and different tag (TAG)
  • docker image push (name of image): pushes image to repository, default is Docker Hub (must log in first)
  • docker image tag (name of image) (name of image):(tag): adds a tag to an image
  • docker image build -t (tag) (directory)
    • -t is for tagging and naming
    • if DockerFile is in current directory, use ".". Otherwise, insert the file path of the DockerFile
